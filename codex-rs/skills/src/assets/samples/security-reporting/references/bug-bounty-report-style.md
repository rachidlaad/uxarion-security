# Bug Bounty Report Style

Use this style only when the user explicitly wants a bug bounty style report or submission draft.

## Tone

- Direct and reproducible.
- Prioritize the minimum information a triager needs to validate the issue.
- Avoid inflated impact claims.

## Writing guidance

- Lead with a short vulnerability summary.
- Include:
  - affected endpoint or target
  - exact reproduction steps
  - payloads or request details when supported by saved artifacts
  - observed impact
  - limitations or missing prerequisites
- Keep remediation brief unless the user explicitly asks for detailed defensive guidance.
- If the issue is not fully confirmed, say that clearly instead of presenting it as a valid bug.
