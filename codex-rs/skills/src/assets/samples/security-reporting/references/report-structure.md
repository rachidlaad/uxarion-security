# Report Structure

Use this baseline structure unless the user explicitly asks for a different format.

## Full session report

1. `# Security Assessment Report`
2. `## Scope`
3. `## Executive Summary`
4. `## Findings`
5. `## Evidence`
6. `## Notes` if needed

## Single finding report

1. `# Finding Report`
2. `## Title`
3. `## Target`
4. `## Severity`
5. `## Summary`
6. `## Reproduction`
7. `## Impact`
8. `## Evidence`
9. `## Remediation`

## Requirements

- Use stable finding IDs when available.
- Reference evidence files by path when useful.
- If the evidence is weak or partial, say so directly.
- Keep Markdown simple and portable.
- Do not include sections that have no support in the saved artifacts unless the user explicitly asks for placeholders.
